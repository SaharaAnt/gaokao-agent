# 基础验收清单 v1（高考作文小助手）

## 目标
先确保“稳定可用”，再做功能升级。每次发布前至少跑一遍本清单。

## A. 启动与加载
- [ ] 使用带版本号地址打开：`/index.html?v=数字`
- [ ] 页面无白屏、无乱码
- [ ] 首页关键区域全部可见（工作台、时间线、范文列表）

## B. 按钮响应（核心）
- [ ] `分析题目` 可输出结果
- [ ] `一键生成800-850字范文` 可写入草稿框
- [ ] `防跑题检查` 有报告
- [ ] `草稿评分` 有分档和维度分
- [ ] `一键提分改写` 可改写并复评分
- [ ] `基础自检` 可输出通过/失败报告

## C. 文本质量
- [ ] 范文字数在 800-850 区间（允许 ±10 浮动）
- [ ] 无“补充论证点1/2/3”之类机械重复
- [ ] 题眼一致，不偷换核心概念
- [ ] 结尾有边界条件，不绝对化

## D. 数据与缓存
- [ ] `data.js/app.js/quick-generate.js` 版本号一致
- [ ] localStorage 可读写（收藏、训练统计正常）
- [ ] 刷新后功能不丢失

## E. 失败回滚策略
- [ ] 若主逻辑异常，独立生成器仍可工作
- [ ] 若评分器异常，至少保证生成和草稿写入可用
- [ ] 失败时结果区有错误提示，不静默失败

## 通过标准
- 核心按钮 100% 可用
- 字数与题眼两项同时达标
- 基础自检报告为“通过”或“基本通过”
